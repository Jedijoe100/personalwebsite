Thesis - Computing Random Units in Number rings

A number ring is an algebraic structure. It consists of a set of numbers, two operators, addition and multiplication and has a number of axioms. For example, the integers \(\mathbb{Z}\) form a ring with addition and multiplication. A unit is an element in a number ring with a multiplicative inverse, i.e. there is are two elements \(a\) and \(a^{-1}\) in our number ring such that \(aa^{-1}=1\). For example, in the integers, the only units are \(\pm 1\) as \(2\) has the inverse \(1/2\) which is not an integer. Another example is the rational numbers \(\mathbb{Q}\), in the rational numbers every element is a unit (every element is of the form \(a/b\) and that has the inverse \(b/a\)). We can extend a number ring to include more elements by adding roots of irreducible polynomials over the ring (where irreducible means the polynomial cannot be split into smaller polynomials). For example, the polynomial \(f(x)=x^{2}+1\) is irreducible over the real numbers \(\mathbb{R}\) (the roots of the polynomial are \(\pm i\notin \mathbb{R}\)) so we can build the extension \(\mathbb{C} = \mathbb{R}/\langle{f(x)}\rangle = \mathbb{R}(i)\) which is a ring containing the roots of \(f(x)\) and \(\mathbb{R}\). From a number ring we can then construct the ring of integers which is a subring that contains all elements in our ring such that they have integer coefficients.
Thus, the aim for the project is to compute random units in the ring of integers for arbitrary fields. For a ring of integers we know how many units are required to generate the entire set of units. There is an algorithm that computes this entire set. This project aims to modify that algorithm to only find one or two units. The application of this project is to improve the performance in finding Brauer Manin obstructions on curves, basically informing whether there are rational solutions to an equation.

Procedurally generated biome maps

One project that I worked on in high school was a program that aimed for realism and was written in Javascript. The map was built from square tiles with the biome depending on three parameters, humidity, temperature and altitude. The humidity depended on the latitude and the see level of the planet. The Temperature depended on the latitude and the altitude. Both of these were varied using perlin noise. Finally, the Altitude was generated using perlin noise. More recently I have developed a python version that allows for easier modification of the biomes using a csv file for the types of biomes and using matrices for the calculations.

Physics Simulation

Early at university I became interested in simulating formation of stellar systems only using gravitational interactions. I would start with about 1000 objects with one object in the center with larger mass than all the other objects. They would all have an initial velocity (as otherwise they would all collapse into a single object). Then all I would do is simulate each frame checking for collisions and computing movement with the only force being gravity.
